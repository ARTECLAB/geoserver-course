🖥️ ¿Qué es GeoServer?

Un servidor que sirve datos geográficos por internet

GeoServer es un programa de código abierto que se instala en un servidor (tu PC, un servidor de la oficina, o la nube). Su trabajo es tomar datos geográficos y exponerlos como servicios web que cualquier aplicación puede consumir: sitios web, apps móviles, QGIS, ArcGIS, o cualquier herramienta que entienda los estándares OGC.

Piensa en un restaurante:

🗄️
Tus datos
Shapefiles, PostGIS, GeoTIFF = ingredientes crudos
👨‍🍳
GeoServer
El chef que procesa, estiliza y sirve los datos
📋
WMS / WFS
El menú: estándares para pedir datos
🌐📱
Clientes
Web, móvil, QGIS = los comensales
💡
No mandas datos crudos al cliente. GeoServer los procesa (renderiza, filtra, estiliza) y sirve como imágenes (WMS) o como datos estructurados (WFS). El cliente nunca toca tus archivos originales.
Los 3 estándares OGC que dominarás

OGC (Open Geospatial Consortium) define los estándares internacionales para que servidores y clientes GIS hablen el mismo idioma:

WMS — Web Map Service
Pides una imagen del mapa → recibes un PNG. Es lo más común. El servidor renderiza los datos y te manda la foto.
WFS — Web Feature Service
Pides los datos vectoriales → recibes GeoJSON o GML. El cliente puede analizar, filtrar y estilizar los datos por su cuenta.
WCS — Web Coverage Service
Pides datos raster → recibes la imagen satelital o DEM completa. Menos común, más especializado.

En este curso nos enfocamos en WMS y WFS — son los que usarás en el 95% de los proyectos reales.

📦 Instalación de GeoServer

Requisitos previos
Java 11+ — GeoServer está escrito en Java. Sin Java, no arranca.
2 GB RAM — mínimo libre para que GeoServer funcione bien.
Puerto 8080 — debe estar libre. Si otro programa lo usa, cámbialo.
Windows — Instalación con binario ZIP
PowerShell
# 1. Verificar que Java está instalado
java -version

# 2. Descargar GeoServer desde:
#    https://geoserver.org/release/stable/
#    → "Binary (OS independent)" → descargar ZIP

# 3. Descomprimir en C:\geoserver

# 4. Iniciar GeoServer
cd C:\geoserver\bin
startup.bat

# 5. Esperar ~30 segundos y abrir en el navegador:
#    http://localhost:8080/geoserver
Linux (Ubuntu/Debian)
Bash
# 1. Instalar Java
sudo apt update
sudo apt install openjdk-11-jdk -y
java -version

# 2. Descargar y descomprimir GeoServer
cd /opt
sudo wget https://sourceforge.net/projects/geoserver/files/GeoServer/2.26.1/geoserver-2.26.1-bin.zip
sudo unzip geoserver-2.26.1-bin.zip
sudo mv geoserver-2.26.1 geoserver

# 3. Iniciar
cd /opt/geoserver/bin
sudo ./startup.sh

# 4. Abrir: http://localhost:8080/geoserver

🧭 La Interfaz de Administración

Las 5 secciones que necesitas conocer ahora
Server Status¿Está vivo el servidor? Memoria disponible, versión instalada, estado general.
WorkspacesCarpetas organizativas. Agrupan capas relacionadas (ej: un workspace por proyecto).
StoresConexiones a fuentes de datos. Le dicen a GeoServer DÓNDE están tus datos.
LayersCapas publicadas. Cada capa es un dataset accesible vía WMS/WFS.
Layer PreviewVista previa rápida. Verificas que tu capa se ve correctamente antes de usarla.
La jerarquía fundamental de GeoServer

Todo en GeoServer sigue esta estructura. Si la entiendes, entiendes GeoServer:

Jerarquía
📁 Workspace  (carpeta organizativa — ej: "mi_proyecto")
 └── 🔌 Store  (conexión a datos — ej: "archivo shapefile" o "base PostGIS")
      └── 🗺️ Layer  (capa publicada — ej: "países")
           └── 🎨 Style  (estilo visual — ej: "polígonos verdes con borde")
💡
Piensa en QGIS: un proyecto (.qgz) contiene conexiones a datos (archivos, bases), que exponen capas, cada una con su estilo. GeoServer es la misma lógica pero del lado del servidor.

🚀 Publica Tu Primera Capa

Paso 1 — Descargar datos de práctica

Descarga un shapefile de países del mundo desde Natural Earth (datos gratuitos, sin licencia):

URL
https://www.naturalearthdata.com/downloads/110m-cultural-vectors/

Descarga "Admin 0 – Countries". Descomprime el ZIP en una carpeta accesible (ej: C:\geodatos\ o /home/tu_usuario/geodatos/).

Paso 2 — Crear un Workspace

Menú lateral → Workspaces → Add new workspace

Configuración
Name:       mi_proyecto
Namespace:  http://localhost/mi_proyecto
Default:    ✓ (marcado)
💡
¿Qué es el Namespace? Es un identificador único en formato URL. No necesita ser una URL real — es solo para que GeoServer diferencie workspaces. El nombre es lo que usarás en las URLs de tus servicios.
Paso 3 — Crear un Store (conexión al shapefile)

Menú lateral → Stores → Add new Store → Shapefile

Configuración
Workspace:       mi_proyecto
Data Source Name: paises_shp
Shapefile URL:   file:///C:/geodatos/ne_110m_admin_0_countries.shp
                 (o la ruta donde descomprimiste)
Character Set:   UTF-8
⚠️
La ruta al shapefile debe ser accesible por GeoServer. Si GeoServer corre como servicio, verifica que el usuario del servicio tenga permisos de lectura en esa carpeta.
Paso 4 — Publicar la capa

Después de guardar el Store, GeoServer te muestra las capas disponibles. Haz clic en "Publish" junto al nombre.

En la pestaña Data:

Configuración
Name:     paises
Title:    Países del mundo
Abstract: Límites de países — fuente Natural Earth

SRS:      EPSG:4326
          → Clic en "Compute from data"
          → Clic en "Compute from native bounds"

Clic en Save.

Paso 5 — Verificar en Layer Preview

Menú lateral → Layer Preview → Buscar "paises" → Clic en OpenLayers

Si ves un mapa con los países del mundo, tu primera capa está publicada y funcionando.

🎉
¡Felicidades! Tienes un servidor de mapas real funcionando en tu máquina. Cualquier aplicación que se conecte a http://localhost:8080/geoserver puede consumir tus datos.

🔗 Anatomía de una URL WMS

Tu servidor ya está sirviendo mapas — probémoslo

Copia esta URL y pégala en tu navegador (ajusta el nombre de workspace y capa):

URL WMS
http://localhost:8080/geoserver/mi_proyecto/wms?
  service=WMS
  &version=1.1.1
  &request=GetMap
  &layers=mi_proyecto:paises
  &styles=
  &srs=EPSG:4326
  &bbox=-180,-90,180,90
  &width=800
  &height=400
  &format=image/png

Cada parámetro tiene un propósito:

serviceTipo de servicio: WMS
requestQué pides: GetMap = "dame una imagen del mapa"
layersQué capa quieres: workspace:nombre_capa
bboxZona geográfica: minX,minY,maxX,maxY (west,south,east,north)
width / heightTamaño de la imagen en píxeles
srsSistema de coordenadas: EPSG:4326 = latitud/longitud
formatFormato de la imagen: image/png, image/jpeg

Si ves una imagen PNG con los países, tu servidor WMS está funcionando perfectamente.

📖 Glosario — Clase 01

GeoServerServidor de datos geográficos de código abierto. Publica datos como servicios WMS/WFS.
OGCOpen Geospatial Consortium. Define los estándares internacionales de interoperabilidad GIS.
WMSWeb Map Service. Devuelve imágenes renderizadas del mapa (PNG, JPEG).
WFSWeb Feature Service. Devuelve datos vectoriales (GeoJSON, GML) para análisis.
WorkspaceCarpeta organizativa en GeoServer. Agrupa capas relacionadas.
StoreConexión a una fuente de datos. Le dice a GeoServer dónde están los archivos o la base de datos.
LayerCapa publicada y accesible vía WMS/WFS. Cada Store puede tener múltiples Layers.
Layer PreviewHerramienta de GeoServer para ver rápidamente si una capa se publica correctamente.
EPSG:4326Sistema de coordenadas geográficas WGS84. Latitud/longitud en grados. El más común.
GetMapTipo de petición WMS: "dame una imagen del mapa con estos parámetros".
ShapefileFormato de archivo vectorial (.shp + .shx + .dbf + .prj). Requiere los 4 archivos juntos.
💻 Práctica 01
Tu Primer Servidor de Mapas
  1. Instala GeoServer en tu máquina (Windows o Linux). Verifica que arranca en http://localhost:8080/geoserver.
  2. Descarga un shapefile de países desde Natural Earth (Admin 0 – Countries).
  3. Crea un Workspace llamado mi_proyecto.
  4. Crea un Store tipo Shapefile apuntando al archivo descargado. Charset: UTF-8.
  5. Publica la capa. Pon nombre, título, SRS (EPSG:4326) y computa el Bounding Box.
  6. Verifica en Layer Preview que el mapa se ve correctamente.
  7. Prueba la URL WMS en el navegador — verifica que devuelve una imagen PNG.
  8. Bonus: Agrega la URL WMS de tu GeoServer como capa en QGIS (Capa → Agregar WMS/WMTS).

✅ Checklist — Clase 01

  • GeoServer instalado y corriendo en localhost:8080
  • Login exitoso en la interfaz de administración
  • Conozco las secciones principales de la interfaz
  • Workspace creado
  • Store creado con un shapefile
  • Capa publicada y visible en Layer Preview
  • URL WMS probada en el navegador
  • Entiendo la jerarquía: Workspace → Store → Layer
