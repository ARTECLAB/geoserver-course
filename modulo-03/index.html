🎨 ¿Qué es SLD?

El lenguaje para diseñar mapas en GeoServer

SLD (Styled Layer Descriptor) es el estándar OGC para definir cómo se visualizan los datos geográficos. Es un archivo XML que le dice a GeoServer exactamente cómo dibujar cada feature: colores, grosores, tamaños, etiquetas y reglas condicionales.

💡
Analogía con QGIS: En QGIS cambias la simbología desde Propiedades → Simbología, eligiendo colores y clasificaciones con clics. SLD es lo mismo pero escrito como XML. La ventaja: es portátil, versionable y se aplica del lado del servidor.
Dónde se edita un SLD

En GeoServer: menú lateral → Styles → Add a new style (o editar uno existente). El editor tiene validación, vista previa de leyenda y formato automático.

El ciclo de trabajo es: editar → validar → previsualizar leyenda → guardar → verificar en Layer Preview.

🏗️ Estructura de un SLD

La plantilla base — copia esto y solo cambia las Rules
XML — Plantilla SLD
<?xml version="1.0" encoding="UTF-8"?>
<StyledLayerDescriptor version="1.0.0"
  xmlns="http://www.opengis.net/sld"
  xmlns:ogc="http://www.opengis.net/ogc"
  xmlns:xlink="http://www.w3.org/1999/xlink"
  xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">

  <NamedLayer>
    <Name>mi_estilo</Name>
    <UserStyle>
      <Title>Mi Estilo</Title>
      <FeatureTypeStyle>

        <!-- ═══ TUS REGLAS AQUÍ ═══ -->

      </FeatureTypeStyle>
    </UserStyle>
  </NamedLayer>
</StyledLayerDescriptor>
💡
Este envoltorio siempre es igual. No lo memorices — cópialo. Lo que importa es lo que va dentro de <FeatureTypeStyle>: las reglas y simbolizadores.
La jerarquía SLD simplificada
Estructura
StyledLayerDescriptor     ← Contenedor (siempre igual)
  └── NamedLayer           ← Nombre del estilo
       └── UserStyle       ← Tu estilo personalizado
            └── FeatureTypeStyle  ← Grupo de reglas
                 └── Rule         ← UNA regla: "si cumple X, dibuja Y"
                      ├── Filter       ← Condición (opcional)
                      └── Symbolizer   ← Cómo dibujar

⬡ Estilos para Polígonos

Polígono simple — Un color para todos
XML — Dentro de FeatureTypeStyle
<Rule>
  <Name>paises</Name>
  <Title>Países</Title>
  <PolygonSymbolizer>
    <Fill>
      <CssParameter name="fill">#2d8659</CssParameter>
      <CssParameter name="fill-opacity">0.6</CssParameter>
    </Fill>
    <Stroke>
      <CssParameter name="stroke">#1a5c3a</CssParameter>
      <CssParameter name="stroke-width">1</CssParameter>
    </Stroke>
  </PolygonSymbolizer>
</Rule>
Clasificación por rango — Coroplético de población

Colores diferentes según un valor numérico. Es lo que en QGIS se llama simbología "Graduada":

XML — Ejemplo: 3 rangos de población
<!-- Población alta -->
<Rule>
  <Title>Población > 50 millones</Title>
  <ogc:Filter>
    <ogc:PropertyIsGreaterThan>
      <ogc:PropertyName>pop_est</ogc:PropertyName>
      <ogc:Literal>50000000</ogc:Literal>
    </ogc:PropertyIsGreaterThan>
  </ogc:Filter>
  <PolygonSymbolizer>
    <Fill><CssParameter name="fill">#dc2626</CssParameter></Fill>
    <Stroke><CssParameter name="stroke">#7f1d1d</CssParameter></Stroke>
  </PolygonSymbolizer>
</Rule>

<!-- Población media -->
<Rule>
  <Title>Población 10 - 50 millones</Title>
  <ogc:Filter>
    <ogc:And>
      <ogc:PropertyIsGreaterThanOrEqualTo>
        <ogc:PropertyName>pop_est</ogc:PropertyName>
        <ogc:Literal>10000000</ogc:Literal>
      </ogc:PropertyIsGreaterThanOrEqualTo>
      <ogc:PropertyIsLessThanOrEqualTo>
        <ogc:PropertyName>pop_est</ogc:PropertyName>
        <ogc:Literal>50000000</ogc:Literal>
      </ogc:PropertyIsLessThanOrEqualTo>
    </ogc:And>
  </ogc:Filter>
  <PolygonSymbolizer>
    <Fill><CssParameter name="fill">#f59e0b</CssParameter></Fill>
    <Stroke><CssParameter name="stroke">#92400e</CssParameter></Stroke>
  </PolygonSymbolizer>
</Rule>

<!-- Población baja (el resto) -->
<Rule>
  <Title>Población < 10 millones</Title>
  <ElseFilter/>
  <PolygonSymbolizer>
    <Fill><CssParameter name="fill">#22c55e</CssParameter></Fill>
    <Stroke><CssParameter name="stroke">#166534</CssParameter></Stroke>
  </PolygonSymbolizer>
</Rule>
💡
ElseFilter captura todo lo que no coincidió con las reglas anteriores. Es el "todos los demás". Siempre ponla como última regla.

〰️ Estilos para Líneas

Ríos con grosor por importancia
XML — LineSymbolizer
<!-- Ríos principales -->
<Rule>
  <Title>Ríos principales</Title>
  <ogc:Filter>
    <ogc:PropertyIsEqualTo>
      <ogc:PropertyName>scalerank</ogc:PropertyName>
      <ogc:Literal>1</ogc:Literal>
    </ogc:PropertyIsEqualTo>
  </ogc:Filter>
  <LineSymbolizer>
    <Stroke>
      <CssParameter name="stroke">#3b82f6</CssParameter>
      <CssParameter name="stroke-width">3</CssParameter>
      <CssParameter name="stroke-linecap">round</CssParameter>
    </Stroke>
  </LineSymbolizer>
</Rule>

<!-- Ríos menores (línea punteada) -->
<Rule>
  <Title>Ríos menores</Title>
  <ElseFilter/>
  <LineSymbolizer>
    <Stroke>
      <CssParameter name="stroke">#93c5fd</CssParameter>
      <CssParameter name="stroke-width">1</CssParameter>
      <CssParameter name="stroke-dasharray">4 3</CssParameter>
    </Stroke>
  </LineSymbolizer>
</Rule>
💡
stroke-dasharray crea líneas punteadas. 4 3 significa "4 píxeles de línea, 3 de espacio". Úsalo para ríos menores, límites provisionales, etc.

● Estilos para Puntos

Ciudades con tamaño proporcional
XML — PointSymbolizer
<Rule>
  <Title>Ciudades grandes</Title>
  <ogc:Filter>
    <ogc:PropertyIsGreaterThan>
      <ogc:PropertyName>pop_max</ogc:PropertyName>
      <ogc:Literal>5000000</ogc:Literal>
    </ogc:PropertyIsGreaterThan>
  </ogc:Filter>
  <PointSymbolizer>
    <Graphic>
      <Mark>
        <WellKnownName>circle</WellKnownName>
        <Fill><CssParameter name="fill">#dc2626</CssParameter></Fill>
        <Stroke><CssParameter name="stroke">#fff</CssParameter></Stroke>
      </Mark>
      <Size>14</Size>
    </Graphic>
  </PointSymbolizer>
</Rule>

Formas disponibles para WellKnownName:

circle● Círculo — ciudades, puntos genéricos
square■ Cuadrado — estaciones, edificios
triangle▲ Triángulo — alertas, picos
star★ Estrella — puntos de interés
cross✚ Cruz — muestras, puntos de control

🏷️ Etiquetas (Labels)

Nombres con halo para legibilidad

Un mapa sin etiquetas está incompleto. Pero etiquetas sin halo son ilegibles sobre fondos de color.

XML — TextSymbolizer
<TextSymbolizer>
  <Label>
    <ogc:PropertyName>name</ogc:PropertyName>
  </Label>
  <Font>
    <CssParameter name="font-family">Arial</CssParameter>
    <CssParameter name="font-size">12</CssParameter>
    <CssParameter name="font-weight">bold</CssParameter>
  </Font>
  <Fill>
    <CssParameter name="fill">#1e293b</CssParameter>
  </Fill>
  <!-- Halo blanco = texto legible sobre cualquier fondo -->
  <Halo>
    <Radius>2</Radius>
    <Fill><CssParameter name="fill">#ffffff</CssParameter></Fill>
  </Halo>
  <!-- Evitar que las etiquetas se superpongan -->
  <VendorOption name="conflictResolution">true</VendorOption>
</TextSymbolizer>
⚠️
Siempre agrega Halo y conflictResolution. Sin halo, el texto se pierde sobre colores. Sin conflictResolution, las etiquetas se superponen y el mapa es ilegible.

🔍 Escalas — Mostrar/ocultar por zoom

Controlar qué se ve según el nivel de zoom

No tiene sentido mostrar el nombre de cada ciudad cuando ves el mundo entero. Las escalas controlan qué se muestra en cada nivel de zoom:

XML — Etiquetas solo en zoom cercano
<Rule>
  <!-- Solo visible entre estas escalas -->
  <MinScaleDenominator>1</MinScaleDenominator>
  <MaxScaleDenominator>5000000</MaxScaleDenominator>
  <TextSymbolizer>
    <!-- etiqueta... -->
  </TextSymbolizer>
</Rule>
💡
Referencia rápida de escalas: País completo ≈ 10,000,000+. Región ≈ 1,000,000. Ciudad ≈ 50,000-500,000. Barrio ≈ 5,000-50,000.

📖 Glosario — Clase 04

SLDStyled Layer Descriptor. Estándar OGC (XML) para definir cómo se visualizan los datos.
RuleUna regla SLD: "si el dato cumple esta condición, dibújalo así".
FilterCondición que determina a qué features aplica la regla (por atributo, rango, etc.).
ElseFilter"Todos los demás" — captura lo que no coincidió con ninguna regla anterior.
PolygonSymbolizerSimbolizador para polígonos: relleno (Fill) + borde (Stroke).
LineSymbolizerSimbolizador para líneas: color, grosor, estilo (sólida, punteada).
PointSymbolizerSimbolizador para puntos: forma (circle, square, star), tamaño, color.
TextSymbolizerSimbolizador para etiquetas: fuente, tamaño, color, halo, posición.
HaloContorno alrededor del texto que mejora la legibilidad sobre fondos de color.
ScaleDenominatorControla en qué nivel de zoom se muestra una regla. Min = cercano, Max = lejano.
GetLegendGraphicPetición WMS que genera la imagen de la leyenda automáticamente desde las reglas SLD.
💻 Práctica 04
Estilos SLD Profesionales
  1. Crea un estilo SLD para polígonos (países) con clasificación por población: al menos 3 rangos con colores diferentes.
  2. Crea un estilo SLD para líneas (ríos) con grosor diferente según importancia (scalerank o longitud).
  3. Crea un estilo SLD para puntos (ciudades) con tamaño proporcional a la población.
  4. Agrega etiquetas con halo a la capa de países o ciudades.
  5. Aplica control de escala: etiquetas visibles solo al hacer zoom cercano.
  6. Asigna cada estilo como Default Style de su capa respectiva.
  7. Verifica en Layer Preview que todo se ve correctamente.
  8. Prueba la leyenda generada: http://localhost:8080/geoserver/wms?service=WMS&request=GetLegendGraphic&layer=mi_proyecto:paises&format=image/png
🚀 Reto Extra — Carreteras con casing
Efecto doble línea

Crea un estilo con dos Rules para la misma capa de líneas: una línea gruesa oscura de fondo (5px, #374151) y una línea delgada clara encima (3px, #fbbf24). El resultado parece una carretera con borde. Es el mismo truco que usa Google Maps.
