📂 Configuración Detallada de una Capa

Más allá del mínimo — cómo lo haría un profesional

En la Clase 01 publicamos una capa con la configuración mínima. Ahora vamos a hacerlo correctamente, como lo esperaría un cliente o una institución:

Pestaña Data — Configuración completa
Name:     paises
          → Nombre técnico: minúsculas, sin espacios ni tildes
          → Es lo que aparece en las URLs de tus servicios

Title:    Países del Mundo
          → Nombre legible para humanos
          → Es lo que ven los usuarios en QGIS al agregar WMS

Abstract: Límites de países del mundo.
          Fuente: Natural Earth, escala 1:110m, 2024.
          Sistema de referencia: EPSG:4326 (WGS84).
          → Descripción técnica con fuente y fecha

Keywords: paises, limites, administrativo, natural-earth
          → Ayudan a encontrar la capa en catálogos

SRS:      EPSG:4326
Bounding Box: → "Compute from data" → "Compute from native bounds"
💡
Regla de oro de los metadatos: Si mañana alguien ve tu servicio WMS sin poder preguntarte nada, ¿entendería qué datos tiene y de dónde vienen? Si no, tus metadatos son insuficientes.
Reproyección al vuelo

Tu shapefile puede estar en una proyección diferente a la que quieres servir. GeoServer puede reproyectar automáticamente:

Configuración SRS
SRS nativo:     EPSG:32719  (UTM zona 19S — como está el archivo)
SRS declarado:  EPSG:4326   (WGS84 — como quieres servirlo)
Acción:         Reproject native to declared
→ GeoServer reproyecta al vuelo sin modificar el archivo original

📁 Directory Store — Muchos shapefiles de una vez

¿Qué es un Directory Store?

En vez de crear un Store por cada shapefile, apuntas a una carpeta entera. GeoServer detecta todos los shapefiles y te permite publicar cada uno:

Stores → Add new Store → Directory of spatial files
Workspace:  mi_proyecto
Store name: natural_earth_data
URL:        file:///C:/geodatos/natural_earth/
Charset:    UTF-8
💡
Ventaja: Si después agregas un nuevo shapefile a la carpeta, solo tienes que ir a Layers → Add new layer → seleccionar el Store → y publicar. No necesitas crear un Store nuevo.

⚠️ Limitaciones del Shapefile

Lo que debes saber antes de usarlos en producción
Campos max 10 charssuperficie_km2 se trunca a superfici. Pierdes contexto en los nombres.
Sin valores nulosCampos vacíos se llenan con 0 o espacios en blanco. No puedes distinguir "sin dato" de "cero".
Una geometría por archivoNo puedes mezclar puntos y polígonos en el mismo shapefile.
Tamaño max ~2 GBPara datos grandes (millones de registros), el shapefile no alcanza.
Encoding frágilTildes y caracteres especiales se corrompen si el charset no es UTF-8.
⚠️
El shapefile es un formato de los años 90. Para proyectos nuevos usa PostGIS o GeoPackage. Pero vas a encontrar shapefiles en TODOS los proyectos existentes — saber publicarlos es obligatorio.

🔍 Verificar con GetCapabilities

Ver tus metadatos como los ve el mundo

GetCapabilities es la petición WMS que devuelve el catálogo completo de tu servidor. Pega esta URL en el navegador:

URL
http://localhost:8080/geoserver/mi_proyecto/wms?service=WMS&version=1.1.1&request=GetCapabilities

Busca tus capas en el XML. Deberías ver el Title, Abstract y Keywords que configuraste. Eso es lo que lee QGIS cuando un usuario agrega tu servicio WMS.

AbstractDescripción de la capa: qué contiene, fuente, fecha, sistema de referencia.
GetCapabilitiesPetición WMS que devuelve el catálogo completo del servidor con todas las capas y metadatos.

SRS declaradoEl sistema de coordenadas en que GeoServer sirve el dato (como lo ve el cliente).

🎯 Objetivo

Conectar GeoServer a una base de datos PostGIS, publicar capas desde tablas, crear vistas SQL dinámicas y organizar todo en grupos de capas.

🐘 ¿Por qué PostGIS?

Shapefile vs PostGIS — cuándo usar cada uno
Shapefile
Campos max 10 chars, ~2 GB max, sin SQL, una geometría por archivo, sin edición concurrente. Bueno para datos estáticos simples.
PostGIS
Sin límite de campos, sin límite de tamaño, SQL completo, vistas dinámicas, edición concurrente, índices espaciales eficientes.
💡
Regla práctica: Para datos estáticos simples → Shapefile. Para cualquier otra cosa → PostGIS. En producción real, PostGIS es casi siempre la mejor opción.

🔧 Preparar PostGIS

Crear la base de datos
SQL — psql o PgAdmin
-- Crear base de datos
CREATE DATABASE geoserver_curso;

-- Conectar
\c geoserver_curso

-- Activar extensión PostGIS
CREATE EXTENSION postgis;

-- Verificar
SELECT PostGIS_Version();
Importar shapefiles a PostGIS

Opción A — Línea de comandos (shp2pgsql):

Bash
shp2pgsql -s 4326 -I ne_110m_admin_0_countries.shp public.paises | psql -d geoserver_curso

Opción B — Desde QGIS:

Capa → Clic derecho → Exportar → Guardar como → Formato: PostgreSQL → tu conexión → nombre de tabla.

💡
El flag -I en shp2pgsql crea un índice espacial automáticamente. Sin índice, las consultas WMS serán mucho más lentas. Siempre usa -I.

🔌 Conectar GeoServer a PostGIS

Crear Store tipo PostGIS

Menú lateral → Stores → Add new Store → PostGIS

Configuración
Workspace:  mi_proyecto
Store name: postgis_datos

Connection Parameters:
  host:     localhost
  port:     5432
  database: geoserver_curso
  schema:   public
  user:     postgres
  passwd:   tu_contraseña

  Validate connections: ✓
⚠️
Si falla la conexión: Antes de buscar el error en GeoServer, verifica que puedes conectarte a la base desde psql o PgAdmin. Si no funciona ahí, no va a funcionar en GeoServer. Causas comunes: PostgreSQL no está corriendo, contraseña incorrecta, o pg_hba.conf no permite conexiones locales.

🔮 Vistas SQL — Capas dinámicas sin crear tablas

¿Qué es una Vista SQL?

Una Vista SQL es una capa que no existe como tabla — se genera al vuelo ejecutando una consulta SQL cada vez que alguien la pide. Es una de las funcionalidades más potentes de GeoServer + PostGIS.

Menú lateral → Layers → Add new layer → seleccionar tu Store PostGIS → "Configure new SQL view..."

SQL — Ejemplo: países con más de 50 millones de habitantes
SELECT
  gid,
  name,
  continent,
  pop_est,
  gdp_md_est,
  geom
FROM paises
WHERE pop_est > 50000000
ORDER BY pop_est DESC
💡
¿Por qué es tan potente? Si mañana un país supera los 50 millones de habitantes en tu base de datos, aparece automáticamente en esta capa. No necesitas tocar GeoServer — la consulta SQL se ejecuta en cada petición.

📦 Grupos de Capas (Layer Groups)

Un grupo = una URL, múltiples capas

Un Layer Group agrupa varias capas en un solo servicio WMS. El cliente pide una URL y recibe todas las capas superpuestas.

Menú lateral → Layer Groups → Add new layer group

Configuración
Name:    mapa_base_mundial
Title:   Mapa Base Mundial

Capas (orden de abajo hacia arriba):
  1. paises        (fondo — se dibuja primero)
  2. rios           (encima)
  3. lagos          (encima)
  4. ciudades       (arriba de todo — puntos visibles)
💡
El orden importa. Lo que está primero en la lista se dibuja primero (queda abajo). Los puntos (ciudades) deben ir al final para que se vean sobre los polígonos.

📖 Glosario — Clase 03

PostGISExtensión de PostgreSQL que agrega soporte para datos geográficos. La base de datos espacial más usada del mundo.
shp2pgsqlHerramienta que convierte un shapefile a comandos SQL para importar a PostGIS.
Índice espacialEstructura que acelera las consultas geográficas. Sin índice, las consultas WMS son 10-100x más lentas.
Vista SQLCapa virtual que se genera al vuelo desde una consulta SQL. No existe como tabla — es dinámica.
Layer GroupAgrupación de capas en un solo servicio. El cliente pide una URL y recibe todas las capas superpuestas.
pg_hba.confArchivo de PostgreSQL que controla quién puede conectarse. Si GeoServer no conecta, este archivo suele ser la causa.
schemaEspacio de nombres dentro de PostgreSQL. Por defecto es public. Organiza tablas dentro de la base.
💻 Práctica 03
PostGIS y Grupos de Capas
  1. Crea la base de datos geoserver_curso con la extensión PostGIS activada.
  2. Importa al menos 2 shapefiles de Natural Earth a PostGIS (con shp2pgsql o desde QGIS).
  3. Crea un Store PostGIS en GeoServer y conecta a la base.
  4. Publica al menos 2 capas desde las tablas de PostGIS.
  5. Crea una Vista SQL que filtre datos (ej: solo países de un continente o con población mayor a X).
  6. Crea un Layer Group que combine 3+ capas con orden lógico.
  7. Verifica el Layer Group en Layer Preview.
🚀 Reto Extra — Vista SQL con JOIN
Combinar datos de dos tablas

Si tienes dos tablas en PostGIS (ej: paises con geometría y indicadores con datos económicos), crea una Vista SQL que haga JOIN entre ambas y publique la capa combinada.
