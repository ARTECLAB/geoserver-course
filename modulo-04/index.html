🗺️ WMS a Fondo — Las 4 Operaciones

1. GetCapabilities — "¿Qué tienes disponible?"

La primera petición que hace cualquier cliente. Devuelve un XML con el catálogo completo de tu servidor: capas, formatos, SRS soportados.

URL
http://localhost:8080/geoserver/mi_proyecto/wms?service=WMS&version=1.1.1&request=GetCapabilities
💡
Cuando agregas un WMS en QGIS, internamente hace GetCapabilities para listar las capas. Si los metadatos están vacíos, el usuario ve nombres técnicos sin contexto. Por eso los metadatos del Módulo 2 son tan importantes.
2. GetMap — "Dame una imagen del mapa"

La operación principal. Recibe parámetros y devuelve una imagen:

URL — GetMap
http://localhost:8080/geoserver/mi_proyecto/wms?
  service=WMS&version=1.1.1&request=GetMap
  &layers=mi_proyecto:paises
  &styles=
  &srs=EPSG:4326
  &bbox=-180,-90,180,90
  &width=800&height=400
  &format=image/png
  &transparent=true

Parámetros que puedes modificar:

formatimage/png (transparente), image/jpeg (comprimido), image/png8 (PNG liviano).
transparenttrue = fondo transparente. false = fondo blanco (o bgcolor).
width / heightTamaño en píxeles de la imagen. Más grande = más detalle pero más pesado.
layersPuedes pedir varias capas separadas por coma. Se superponen en orden.
💡
Truco: Pide múltiples capas en una sola imagen: &layers=mi_proyecto:paises,mi_proyecto:rios,mi_proyecto:ciudades. Se dibujan en orden (primero abajo, último arriba).
3. GetFeatureInfo — "¿Qué hay en este punto?"

Consulta los atributos del feature en una posición del mapa. Es el "Identificar" de QGIS pero desde una URL. Devuelve JSON, HTML o GML.

URL — GetFeatureInfo
http://localhost:8080/geoserver/mi_proyecto/wms?
  service=WMS&version=1.1.1&request=GetFeatureInfo
  &layers=mi_proyecto:paises
  &query_layers=mi_proyecto:paises
  &info_format=application/json
  &srs=EPSG:4326&bbox=-180,-90,180,90
  &width=800&height=400
  &x=400&y=200
  &feature_count=5
⚠️
Para que GetFeatureInfo funcione, la capa debe tener Queryable activado en la pestaña Publishing de la configuración de la capa.
4. GetLegendGraphic — "Dame la leyenda"

Genera una imagen de la leyenda basada en tu estilo SLD:

URL — Leyenda
http://localhost:8080/geoserver/wms?service=WMS&version=1.1.1
  &request=GetLegendGraphic
  &layer=mi_proyecto:paises
  &format=image/png
  &LEGEND_OPTIONS=fontName:Arial;fontSize:12;fontAntiAliasing:true
💡
La leyenda se genera automáticamente desde los <Title> de cada Rule de tu SLD. Por eso es importante que cada Rule tenga un Title descriptivo.

📦 WFS — Acceso a los Datos

¿WMS o WFS? Cuándo usar cada uno
WMS — Imagen
Visualizar mapas. Recibir fotos. Liviano. El 90% del tiempo.
WFS — Datos
Descargar datos. Analizar. Filtrar. Exportar a Shapefile/GeoJSON.
GetFeature — Descargar datos
URL — GeoJSON
http://localhost:8080/geoserver/mi_proyecto/wfs?
  service=WFS&version=2.0.0&request=GetFeature
  &typeName=mi_proyecto:paises
  &outputFormat=application/json
  &count=10
  &sortBy=pop_est+D

Formatos de descarga disponibles:

application/jsonGeoJSON — para apps web y móviles.
SHAPE-ZIPDescarga un ZIP con shapefile listo para QGIS. El servidor lo genera al vuelo.
csvCSV para abrir en Excel o Python.
KMLPara Google Earth.
🎯
SHAPE-ZIP es un servicio de descarga automático. Tus usuarios pueden descargar shapefiles directamente desde una URL sin que tú hagas nada. GeoServer genera el ZIP al vuelo.
DescribeFeatureType — Ver la estructura de campos

Antes de escribir filtros CQL, necesitas saber los nombres exactos de los campos:

URL
http://localhost:8080/geoserver/mi_proyecto/wfs?
  service=WFS&version=2.0.0&request=DescribeFeatureType
  &typeName=mi_proyecto:paises
⚠️
Los nombres de campos son case-sensitive en PostGIS. Si el campo se llama pop_est, escribir Pop_Est en un filtro CQL no funcionará. Siempre verifica con DescribeFeatureType.

⚡ Filtros CQL — Consultas Dinámicas

¿Qué es CQL?

CQL (Common Query Language) es un lenguaje para filtrar datos directamente desde la URL. Es como un WHERE de SQL que se envía como parámetro HTTP. El servidor filtra ANTES de enviar — más eficiente que filtrar en el cliente.

Funciona tanto en WMS (GetMap) como en WFS (GetFeature). Solo agregas &CQL_FILTER=... a la URL.

Filtros por texto
CQL
# Exacto
CQL_FILTER=name='Brazil'

# Contiene
CQL_FILTER=name LIKE '%land%'

# Empieza con
CQL_FILTER=name LIKE 'Arg%'

# En una lista
CQL_FILTER=name IN ('Brazil','Argentina','Chile')
Filtros por valor numérico
CQL
# Mayor que
CQL_FILTER=pop_est > 100000000

# Rango
CQL_FILTER=pop_est BETWEEN 10000000 AND 50000000
Operadores lógicos: AND, OR, NOT
CQL
# Ambas condiciones
CQL_FILTER=continent='South America' AND pop_est > 10000000

# Al menos una
CQL_FILTER=continent='Europe' OR continent='Africa'

# Excluir
CQL_FILTER=NOT (continent='Antarctica')

# Combinados
CQL_FILTER=(continent='South America' OR continent='Africa') AND pop_est > 50000000
Filtros espaciales
CQL
# Dentro de un rectángulo
CQL_FILTER=BBOX(geom, -80, -60, -30, 15)

# Dentro de un radio (50 km desde un punto)
CQL_FILTER=DWITHIN(geom, POINT(-58.38 -34.60), 50000, meters)
⚠️
En CQL el orden es POINT(longitud latitud) — longitud primero, luego latitud. Es diferente a LatLng. Y el nombre del campo de geometría debe coincidir exactamente con tu tabla (geom, the_geom, etc.).
Ejemplo completo: Filtrar + Descargar

Descargar como shapefile todos los países de Sudamérica con más de 10 millones de habitantes:

URL completa
http://localhost:8080/geoserver/mi_proyecto/wfs?
  service=WFS&version=2.0.0&request=GetFeature
  &typeName=mi_proyecto:paises
  &outputFormat=SHAPE-ZIP
  &CQL_FILTER=continent='South America' AND pop_est>10000000
  &propertyName=name,continent,pop_est,geom

Pega esta URL en el navegador → descarga un ZIP → ábrelo en QGIS → verás solo Brasil, Argentina, Colombia, Perú y Venezuela.

📖 Glosario — Clase 05

GetCapabilitiesPetición que devuelve el catálogo completo del servidor: capas, formatos, SRS.
GetMapPetición WMS que devuelve una imagen renderizada del mapa.
GetFeatureInfoPetición WMS que devuelve atributos del feature en un punto del mapa.
GetLegendGraphicPetición WMS que genera la imagen de la leyenda desde el SLD.
GetFeaturePetición WFS que devuelve datos vectoriales (GeoJSON, Shapefile, CSV).
DescribeFeatureTypePetición WFS que devuelve la estructura de campos de una capa.
CQL_FILTERParámetro que envía un filtro CQL al servidor. Funciona en WMS y WFS.
CQLCommon Query Language. Lenguaje simple para filtrar datos desde la URL.
LIKEOperador CQL para buscar patrones en texto. % = cualquier texto.
DWITHINFiltro CQL espacial: features dentro de un radio desde un punto.
SHAPE-ZIPFormato de salida WFS que descarga un shapefile como ZIP.
LEGEND_OPTIONSParámetros para personalizar la leyenda: fuente, tamaño, color.
💻 Práctica 05
WMS, WFS & CQL
  1. Prueba GetCapabilities de WMS y WFS en el navegador. Busca tus capas en el XML.
  2. Construye una URL GetMap con 2+ capas superpuestas. Prueba diferentes formatos (PNG, JPEG).
  3. Prueba GetFeatureInfo en Layer Preview. Cambia info_format a application/json y analiza la respuesta.
  4. Genera la leyenda con GetLegendGraphic y personaliza con LEGEND_OPTIONS.
  5. Descarga una capa como GeoJSON y como SHAPE-ZIP.
  6. Aplica al menos 6 filtros CQL diferentes: texto exacto, LIKE, numérico, BETWEEN, AND/OR, IN, y uno espacial (BBOX o DWITHIN).
  7. Conecta tu GeoServer desde QGIS como WMS y como WFS.
  8. Aplica un filtro CQL desde QGIS sobre una capa WFS.
