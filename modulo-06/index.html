🌐 Parte 1 — Visor Web con Leaflet

¿Qué es Leaflet?

Leaflet es la librería JavaScript más popular para crear mapas interactivos en la web. Es liviana (~42KB), fácil de aprender, de código abierto y funciona en todos los navegadores. La usan miles de sitios web: desde el New York Times hasta gobiernos de todo el mundo.

Para este módulo usamos HTML puro + Leaflet — no necesitas frameworks ni herramientas de compilación. Un solo archivo HTML que puedes abrir directamente en el navegador.

Visor básico con mapa base

Crea un archivo visor.html y pega este código:

HTML — visor.html

  </script>

  </style>

  <div id="mapa"></div>

    // 1. Crear el mapa centrado en Sudamérica
    var mapa = L.map('mapa').setView([-15, -60], 4);

    L.tileLayer('https://tile.openstreetmap.org/{z}/{x}/{y}.png', {

Abre este archivo en el navegador. Deberías ver un mapa interactivo con OpenStreetMap. Este es tu punto de partida.

Agregar tu capa WMS de GeoServer

Ahora conectamos Leaflet a TU GeoServer. Agrega estas líneas dentro del <script>, después del tileLayer de OSM:

JavaScript — Dentro del script
// 3. Agregar capa WMS de tu GeoServer
var capaWMS = L.tileLayer.wms(
  'http://localhost:8080/geoserver/mi_proyecto/wms', {
    layers: 'mi_proyecto:paises',
    format: 'image/png',
    transparent: true,
    version: '1.1.1',
    attribution: 'Datos: mi GeoServer'
  }
).addTo(mapa);
💡
¿Qué hace L.tileLayer.wms()? Le dice a Leaflet: "hay un servidor WMS en esta URL que tiene una capa llamada 'paises'. Pídele las imágenes y muéstralas sobre el mapa base." Leaflet construye las URLs GetMap automáticamente según el zoom y la posición.
Selector de capas WMS

Leaflet tiene un control de capas integrado para activar/desactivar capas:

JavaScript
// Definir varias capas WMS
var paises = L.tileLayer.wms(
  'http://localhost:8080/geoserver/mi_proyecto/wms', {
    layers: 'mi_proyecto:paises',
    format: 'image/png', transparent: true
  });

var rios = L.tileLayer.wms(
  'http://localhost:8080/geoserver/mi_proyecto/wms', {
    layers: 'mi_proyecto:rios',
    format: 'image/png', transparent: true
  });

var ciudades = L.tileLayer.wms(
  'http://localhost:8080/geoserver/mi_proyecto/wms', {
    layers: 'mi_proyecto:ciudades',
    format: 'image/png', transparent: true
  });

// Agregar control de capas
var overlays = {
  "Países": paises,
  "Ríos": rios,
  "Ciudades": ciudades
};

L.control.layers(null, overlays).addTo(mapa);
paises.addTo(mapa); // Activar por defecto
💡
El primer argumento de L.control.layers() son las capas base (radio buttons — solo una activa). El segundo son los overlays (checkboxes — puedes activar varios). Usamos null para base porque ya tenemos OSM fijo.
GetFeatureInfo al hacer clic — Popup con atributos

Cuando el usuario hace clic en el mapa, consultas los atributos del feature con GetFeatureInfo y los muestras en un popup:

JavaScript
// 4. GetFeatureInfo al hacer clic
mapa.on('click', function(e) {
  var latlng = e.latlng;
  var bounds = mapa.getBounds();
  var size = mapa.getSize();
  var point = mapa.latLngToContainerPoint(latlng);

  var url = 'http://localhost:8080/geoserver/mi_proyecto/wms?'
    + 'SERVICE=WMS&VERSION=1.1.1&REQUEST=GetFeatureInfo'
    + '&LAYERS=mi_proyecto:paises'
    + '&QUERY_LAYERS=mi_proyecto:paises'
    + '&INFO_FORMAT=application/json'
    + '&SRS=EPSG:4326'
    + '&WIDTH=' + size.x + '&HEIGHT=' + size.y
    + '&BBOX=' + bounds.toBBoxString()
    + '&X=' + Math.round(point.x)
    + '&Y=' + Math.round(point.y);

  fetch(url)
    .then(function(r) { return r.json(); })
    .then(function(data) {
      if (data.features && data.features.length > 0) {
        var props = data.features[0].properties;
        var html = '<b>' + (props.name || 'Sin nombre') + '</b><br>';
        for (var key in props) {
          if (key !== 'bbox') {
            html += key + ': ' + props[key] + '<br>';
          }
        }
        L.popup()
          .setLatLng(latlng)
          .setContent(html)
          .openOn(mapa);
      }
    });
});
¿Qué es fetch()?

fetch() es la forma moderna de JavaScript para hacer peticiones HTTP. Le das una URL, espera la respuesta del servidor, y cuando llega la procesas. En este caso: pide los atributos a GeoServer (GetFeatureInfo) y cuando responde con JSON, los muestra en un popup de Leaflet.

Filtro CQL desde la interfaz

Agrega un input de texto que permita filtrar la capa WMS dinámicamente:

HTML — Agregar antes del div mapa
<div style="position:absolute; top:10px; left:60px; z-index:1000;
  background:white; padding:8px 12px; border-radius:6px;
  box-shadow:0 2px 6px rgba(0,0,0,0.3);">
  <input id="filtro" placeholder="CQL: continent='South America'"
    style="width:300px; padding:6px;">
  <button onclick="aplicarFiltro()">Filtrar</button>
  <button onclick="limpiarFiltro()">Limpiar</button>
</div>
JavaScript
function aplicarFiltro() {
  var cql = document.getElementById('filtro').value;
  paises.setParams({ CQL_FILTER: cql });
}

function limpiarFiltro() {
  document.getElementById('filtro').value = '';
  paises.setParams({ CQL_FILTER: null });
}
💡
setParams() actualiza los parámetros de la capa WMS y recarga las imágenes automáticamente. No necesitas destruir y recrear la capa — solo cambias el parámetro CQL_FILTER y Leaflet pide nuevas imágenes al servidor.

📱 Parte 2 — Capas WMS en Flutter

¿Cómo se conecta una app móvil a GeoServer?

Exactamente igual que el visor web: la app pide imágenes WMS o datos WFS a GeoServer usando las mismas URLs. La diferencia es que en vez de Leaflet usamos flutter_map, una librería de mapas para Flutter.

💡
No necesitas saber Flutter para entender esto. El código que verás es funcional y puedes copiarlo. Lo importante es entender el concepto: tu GeoServer sirve datos, y tanto la web como el móvil los consumen con las mismas URLs.
Capa WMS en flutter_map

Este es el código clave. Es equivalente al L.tileLayer.wms() de Leaflet:

Dart — Flutter
// Dentro de FlutterMap → children:
TileLayer(
  wmsOptions: WMSTileLayerOptions(
    baseUrl: 'http://tu-servidor:8080/geoserver/mi_proyecto/wms?',
    layers: ['mi_proyecto:paises'],
    transparent: true,
    format: 'image/png',
    version: '1.1.1',
  ),
),
⚠️
En Flutter la URL necesita el ? al final. Flutter_map construye los parámetros WMS después del ?. Sin él, la URL queda malformada. En Leaflet no lo necesitas — lo agrega solo.
Comparación directa: Leaflet vs Flutter
Comparación

                                       ),

La URL WMS es la misma. Solo cambia la librería del cliente.
