⚡ Parte 1 — GeoWebCache: Rendimiento

¿Por qué los mapas pueden ser lentos?

Cada vez que un usuario ve el mapa en tu visor web o app, GeoServer hace lo siguiente:

1. Recibe la petición WMS (GetMap)
2. Lee los datos de la fuente (PostGIS, Shapefile)
3. Aplica el estilo SLD (clasificación, colores, etiquetas)
4. Renderiza la imagen PNG/JPEG
5. Envía la imagen al cliente

Los pasos 2, 3 y 4 consumen CPU y memoria. Si tienes 100 usuarios simultáneos, GeoServer repite este proceso para cada uno. Con datos pesados (millones de polígonos) o estilos complejos (etiquetas + clasificación), el servidor se sobrecarga y los mapas se cargan lento.

¿Qué es GeoWebCache?

GeoWebCache es un sistema de tile caching integrado en GeoServer. La idea es simple: la primera vez que alguien pide un tile, GeoServer lo renderiza y lo guarda en disco. La segunda vez que alguien pide el mismo tile, lo sirve directamente del disco sin renderizar de nuevo.

Sin cache
Cada petición: leer datos → aplicar estilo → renderizar → enviar. ~200-500ms por tile.
Con cache
Primera vez: renderiza y guarda. Después: lee del disco y envía. ~5-20ms por tile.
💡
Es como cocinar en un restaurante. Sin cache: preparas cada plato desde cero cuando lo piden. Con cache: preparas los platos más pedidos por adelantado y los sirves del aparador. El resultado es el mismo pero 10-50x más rápido.
¿Qué es un tile?

Un tile es una imagen cuadrada (generalmente 256×256 píxeles) que representa una porción del mapa en un nivel de zoom específico. El mapa completo se divide en una cuadrícula de tiles.

Concepto
Zoom 0:   1 tile    (el mundo entero)
Zoom 1:   4 tiles   (2×2)
Zoom 2:   16 tiles  (4×4)
Zoom 3:   64 tiles  (8×8)
...
Zoom 10:  ~1 millón de tiles
Zoom 18:  ~69 mil millones de tiles (nivel calle)

Cuando mueves o haces zoom en el mapa, el visor pide solo los tiles visibles. GeoWebCache guarda los que ya se generaron para no recalcularlos.

💡
Google Maps, OpenStreetMap, Mapbox — todos funcionan con tiles precalculados. Es el estándar de la industria para servir mapas rápidamente. GeoWebCache aplica este mismo concepto a tus capas de GeoServer.
Configurar caching en GeoServer

GeoWebCache ya viene integrado en GeoServer 2.28.2. Solo necesitas activar y configurar el caching por capa:

Paso 1 — Verificar que está activo:

URL
http://localhost:8080/geoserver/gwc/

Si ves la interfaz de GeoWebCache, está activo.

Paso 2 — Activar cache por capa:

Menú lateral → Tile Caching → Tile Layers → clic en tu capa → pestaña Tile Caching

Configuración
Enable tile caching:  ✓
Tile image formats:   image/png, image/jpeg
Gridsets:             EPSG:4326, EPSG:900913
Metatiling:           4×4 (genera 16 tiles de una vez — más eficiente)
Gutter:               0

Paso 3 — Pre-generar tiles (seeding):

En vez de esperar a que los usuarios pidan los tiles, puedes precalcularlos:

Tile Caching → Tile Layers → tu capa → Seed / Truncate

Configuración de seed
Number of tasks:  2 (hilos paralelos)
Type of operation: Seed (generar)
Grid set:          EPSG:4326
Format:            image/png
Zoom start:        0
Zoom stop:         10 (no ir más allá sin necesidad)
⚠️
Cuidado con el zoom máximo del seed. Cada nivel de zoom multiplica los tiles por 4. Un seed hasta zoom 18 para una capa de un país entero puede generar millones de tiles y llenar tu disco. Empieza con zoom 10 y sube gradualmente según necesites.
¿Cuándo invalidar el cache?

Si los datos cambian (actualizaste la tabla en PostGIS, cambiaste el estilo SLD), los tiles cacheados quedan obsoletos. Necesitas truncar el cache:

Tile Caching → Tile Layers → tu capa → Seed / Truncate → Type: Truncate

Esto borra los tiles guardados. La próxima vez que alguien pida el mapa, GeoServer regenera los tiles con los datos actualizados.

💡
Regla práctica: Si los datos cambian a diario → no usar cache (o cache de corta duración). Si cambian cada semana/mes → cache + truncate manual cuando actualices. Si son estáticos → cache agresivo con seed completo.

☁️ Parte 2 — Deploy en la Nube

¿Por qué deploy en la nube?

Hasta ahora tu GeoServer corre en localhost — solo tú puedes acceder. Para que otros lo usen (colegas, clientes, el público), necesitas ponerlo en un servidor accesible desde internet.

localhost
Solo tú. Desde tu máquina. Si apagas la PC, el servidor muere.
Servidor en la nube
Cualquiera con la URL. 24/7 disponible. IP pública o dominio propio.
Google Cloud — Free Tier (3 meses gratis)

Google Cloud ofrece $300 USD de crédito gratis por 3 meses para cuentas nuevas. Suficiente para correr un servidor GeoServer pequeño durante el curso y prácticas.

Lo que necesitas:

Cuenta GoogleTu cuenta Gmail normal. Registrarte en cloud.google.com/free
Tarjeta de crédito/débitoGoogle la pide para verificar identidad. NO te cobra mientras estés en el free tier.
VM instanceUna máquina virtual Linux en la nube. Es como tu PC pero en los servidores de Google.
Paso 1 — Crear la máquina virtual

En Google Cloud Console → Compute Engine → VM Instances → Create Instance

Configuración de la VM
Name:           geoserver-curso
Region:         southamerica-east1 (São Paulo — más cercano)
Machine type:   e2-medium (2 vCPU, 4 GB RAM)
Boot disk:      Ubuntu 22.04 LTS, 30 GB SSD
Firewall:       ✓ Allow HTTP traffic
                ✓ Allow HTTPS traffic
💡
e2-medium (4 GB RAM) es suficiente para GeoServer con datos pequeños/medianos. Para producción con muchos usuarios necesitarías más RAM. Pero para el curso y prácticas está perfecto.
Paso 2 — Conectarse por SSH

En la lista de VMs, clic en SSH junto a tu instancia. Se abre una terminal en el navegador.

💡
Google Cloud abre la terminal SSH directamente en el navegador. No necesitas PuTTY ni configurar llaves SSH manualmente.
Paso 3 — Instalar Java + Tomcat + GeoServer

Los mismos pasos que hiciste en tu máquina local en la Clase 1, pero en el servidor cloud:

Bash — En la terminal SSH
# 1. Actualizar sistema
sudo apt update && sudo apt upgrade -y

# 2. Instalar Java
sudo apt install openjdk-11-jdk -y
java -version

# 3. Instalar Tomcat 9
sudo apt install tomcat9 tomcat9-admin -y
sudo systemctl enable tomcat9
sudo systemctl start tomcat9

# 4. Configurar memoria para GeoServer
sudo nano /usr/share/tomcat9/bin/setenv.sh
# Escribir: export CATALINA_OPTS="-Xms512m -Xmx2048m"
# Guardar: Ctrl+O, Enter, Ctrl+X

# 5. Descargar GeoServer 2.28.2 WAR
cd /tmp
wget https://sourceforge.net/projects/geoserver/files/GeoServer/2.28.2/geoserver-2.28.2-war.zip
sudo apt install unzip -y
unzip geoserver-2.28.2-war.zip
sudo cp geoserver.war /var/lib/tomcat9/webapps/

# 6. Reiniciar Tomcat
sudo systemctl restart tomcat9

# 7. Esperar ~60 segundos
# Abrir: http://IP_EXTERNA:8080/geoserver
Paso 4 — Abrir el puerto 8080

Por defecto Google Cloud bloquea puertos que no sean 80 (HTTP) y 443 (HTTPS). Necesitas abrir el 8080:

Google Cloud Console → VPC Network → Firewall → Create Firewall Rule

Configuración firewall
Name:             allow-geoserver
Direction:        Ingress
Target:           All instances
Source IP ranges: 0.0.0.0/0
Protocols/ports:  TCP: 8080
Paso 5 — Acceder desde cualquier parte del mundo

Busca la IP externa de tu VM en la lista de instancias. Abre en el navegador:

URL
http://TU_IP_EXTERNA:8080/geoserver

Si ves la interfaz de GeoServer, tu servidor está en producción. Cualquier persona del mundo puede acceder con esa URL.

⚠️
¡Cambia la contraseña de admin INMEDIATAMENTE! Tu servidor ahora es público. Si dejas admin/geoserver, cualquiera puede entrar. Aplica todo lo del Módulo 5 (seguridad).
Paso 6 — Subir datos al servidor

Opción más simple para copiar shapefiles al servidor:

Bash — Desde tu PC local
# Usando gcloud CLI (instalar desde cloud.google.com/sdk)
gcloud compute scp ./mis_datos/*.shp geoserver-curso:/tmp/ --zone=southamerica-east1-a

# En el servidor SSH, mover a carpeta de datos
sudo mkdir -p /opt/geodatos
sudo mv /tmp/*.shp /opt/geodatos/
sudo chown -R tomcat:tomcat /opt/geodatos

Alternativa: instalar PostGIS en el servidor y conectar remotamente desde PgAdmin.

💻 Práctica 08
Cache + Deploy
  1. Verifica que GeoWebCache está activo en http://localhost:8080/geoserver/gwc/
  2. Activa tile caching para al menos 2 capas de tu GeoServer local.
  3. Ejecuta un seed de zoom 0 a 8 para una capa. Verifica que los tiles se generan.
  4. Compara la velocidad: carga el mapa en Layer Preview sin cache vs con cache. Abre F12 → Network para ver los tiempos.
  5. Crea una cuenta en Google Cloud (cloud.google.com/free).
  6. Crea una VM con Ubuntu 22.04 y 4 GB RAM.
  7. Instala Java + Tomcat + GeoServer en la VM vía SSH.
  8. Abre el puerto 8080 en el firewall de Google Cloud.
  9. Accede a http://TU_IP:8080/geoserver desde tu navegador local.
  10. Cambia la contraseña de admin inmediatamente.
